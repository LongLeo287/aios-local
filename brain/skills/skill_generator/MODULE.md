# Skill-Generator Module — AI OS Integration

## Module Identity

| Property | Value |
|---|---|
| **Module ID** | `skill-generator` |
| **Tier** | T0 — Core OS Foundation |
| **Type** | Meta-Skill (generates other skills) |

| **Runtime** | AI as execution engine (no external CLI) |
| **Source** | `plugins/skill-generator/` |

## Purpose in AI OS

The `skill-generator` is the **self-expansion mechanism** of AI OS.
It allows the system to grow its own capabilities by creating new skills
from user workflows, without requiring manual SKILL.md authoring.

```
User Workflow
     ↓
skill-generator (8-phase pipeline)
     ↓
New SKILL.md → deployed to plugins/ or skills/
     ↓
AI OS gains new capability permanently
```

## Activation

This module is auto-loaded as T0 Core. It activates when:
- User says "tạo skill mới", "create a skill", "automate this"
- User describes a repetitive workflow with >3 steps
- User says "biến quy trình X thành skill"
- User wants to improve an existing skill (Improve Mode)

## Pipeline Reference (8 Phases)

| Phase | Name | Output |
|---|---|---|
| 1 | 🎤 Deep Interview | User workflow extracted |
| 2 | 🔬 Knowledge Extraction | Structured skill blueprint |
| 3 | 🔎 Pattern Detection | Complexity score → architecture |
| 4 | 🏗️ Skill Generation | Complete SKILL.md + package |
| 5 | 🧪 Live Test & Refine | Tested, working skill |
| 6* | 📊 Quantitative Eval | 7-dimension score + CI JSON |
| 7* | 🔄 Iteration Loop | Improved until grade A+ |
| 8* | 🎯 Description Optimization | Trigger accuracy maximized |

*Optional phases, run when skill has measurable output or user requests.

## Complexity Tiers (from Pattern Detection)

| Score | Level | Output Structure |
|---|---|---|
| 1-5 | 🟢 Simple | SKILL.md only |
| 6-12 | 🟡 Medium | SKILL.md + examples/ |
| 13-20 | 🟠 Complex | SKILL.md + resources/ + examples/ |
| 21+ | 🔴 Very Complex | Full: SKILL.md + scripts/ + resources/ |

## Integration with AI OS Tier System

Skills generated by this module are automatically classified:
- **T0**: If the skill becomes a critical dependency
- **T1**: If used in core strategic workflows
- **T2**: Domain-specific operational skills (default for new skills)
- **T3**: Experimental skills still in testing

## Module Files

Full implementation at `plugins/skill-generator/`:
- `SKILL.md` — Main skill definition (388 lines, 8-phase pipeline)
- `phases/` — Detailed phase implementations
- `resources/` — Templates, guides, pattern library
- `examples/` — Output examples
- `scripts/` — Automation scripts
- `agents/` — Specialized sub-agents for skill creation

## Security Constraints

- Generated skills MUST NOT hardcode API keys or secrets
- Destructive scripts require explicit human confirmation
- All generated skills pass through skill_sentry review

## Quality Standards

Skills generated must achieve minimum Grade B on 7-dimension eval:
Correctness • Completeness • Format • Adherence • Safety • Efficiency • Robustness
