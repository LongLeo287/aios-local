# BMAD Method: AI-Driven Development Framework
**Source:** `bmad-code-org/BMAD-METHOD` (Breakthrough Method for Agile AI-Driven Development)  
**Extracted:** 2026-03-14 for AI OS governance

---

## What is BMAD?
An open-source framework that transforms AI from ad-hoc coding assistant into a **disciplined agile participant**. Works with Claude Code, Cursor, GitHub Copilot.

## The 4-Phase Cycle (adopt in AI OS roadmap)
```
Phase 1: ANALYSIS  → Understand problem domain, constraints, existing codebase
Phase 2: PLANNING  → Generate PRD, Architecture Doc, User Stories (as artifacts)
Phase 3: SOLUTIONING → Design system components, API contracts, data models
Phase 4: IMPLEMENTATION → Code, test, deploy — with agents executing validated plans
```

## Specialized Agent Personas (adopt in AI OS AGENTS.md)
| Persona | Responsibility |
|---------|---------------|
| Product Manager | Requirements, user stories, PRD generation |
| Architect | Technical design, system architecture |
| Developer | Implementation, coding |
| QA Engineer | Testing, validation |
| Scrum Master | Sprint planning, task decomposition |
| UX Designer | User experience, interface design |

## Key Innovations to Adopt

### 1. "Party Mode" — Multi-Agent Collaboration
Multiple AI persona agents collaborate in **a single session** for brainstorming or planning. Each agent plays its role and they debate/refine together.
**AI OS Action:** Implement "Party Mode" in Antigravity sessions where complex planning is needed.

### 2. Agentic Planning First
All planning artifacts (PRD, Architecture Doc) are generated by AI agents **before any code is written**. These serve as single source of truth.
**AI OS Action:** Formalize `implementation_plan.md` as mandatory PRD-like artifact.

### 3. Codebase Flattener
A tool that aggregates all project files into a single XML for easy context sharing with AI.
**AI OS Action:** Implement `data_miner.py` to export workspace context as a flat file for cross-AI transfer.

### 4. Scale-Domain-Adaptive
Automatically adjusts planning depth based on project complexity.
- Small project → Lightweight plan
- Enterprise project → Full PRD + Architecture + Sprint breakdown

### 5. Context Engineering (critical!)
Concentrate AI usage in **high-value planning phases** first. Use comprehensive reusable artifacts to reduce token consumption later.
**AI OS Action:** Build artifact libraries for common patterns. Don't reinvent; reference existing artifacts.
